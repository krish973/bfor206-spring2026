# BFOR 206 Lab 9-1: Interviews

## Task Description

Interview the product owner (the instructor) to
gather requirements for your project.

Write your notes in the requirements document.
If you have not yet completed lab 8-2, complete
it while doing this lab.

## Submission instructions

1. Create a new branch for your PR (e.g., `interface-requirements`). See Lab 8-2 for instructions on how to create a branch and open a PR.
2. Add content to the requirements document for your area of the project.
3. Commit your changes with a descriptive message (e.g., "Add names to contributors section").
4. Push your commit to GitHub.
5. Verify that the commit is visible in the
Pull Request created in Lab 8-2.
6. In the PR description, explain the changes you made and how they contribute to the project.
7. Request a review from the instructor `@lspitzley`.
